#### 🎮 **GameObjects** — Игровые объекты
Содержит логику основных игровых сущностей:
- `WarriorModel.cs` — Модель главного строения (замка) с параметрами атаки
- `WarriorView.cs` — Визуальное представление главного строения
- `WarriorController.cs` — Контроллер для управления главным строением
- Другие строения и объекты по паттерну Model-View-Controller

#### 📚 **Registries** — Реестры
Регистрация и кеширование ссылок на игровые объекты
- `LiveRegistry.cs` — находятся все активные живые объекты
- `ProjectileRegistry.cs` — находятся все активные снаряды
- `SaveRegistry.cs` — какие объекты должны быть сохранены, если игрок например во время уровня закрыл аплик

#### 🎮 **ServicesGameplay** — Сервисы игровой логики
Специализированные сервисы для геймплея:
- `TickScheduler.cs` класс отвечающий за распределение задач по кадрам(чтобы не делать все в одном кадре например)
- `DetectionService.cs` Поиск целей, врага
- `MoveAllMovablesService.cs` Говорит что кому и когда нужно идти
- `BlobShadowRotateService.cs` просто чтобы тени всегда поворачивались относительно пола, так как земля не ровная
- `AttackAllLiveService.cs` система говорит когда атаковать
- `ProjectileAttackService.cs` система полета снарядов

#### 🎯 **Services** — Сервисы приложения
Глобальные сервисы, отвечающие за функциональность:
- `SettingsService.cs` Управление звуком и музыкой
- `SaveLoadLevelService.cs` Работа с сохранениями
- `SceneCreator.cs` Конфигурация и настройки игры
- `PoolsManager.cs`  управление всеми пулами

#### 🖼️ **UI** — Интерфейс пользователя
Окна и элементы интерфейса:
- `Windows.cs` — все окна которые будет видеть игрок(рисовал и UX/UI сам)
- `WindowElements.cs` — элементы в самих окнах
- `TweenFeature.cs` — DoTween анимации на основе компонентов, устаревшая версия тут(чтобы разгрузить логику окон, анимации можно сделать так)

#### 📊 **AllAppData** — Глобальные данные приложения
Хранилище состояния игры:
- `AppData.cs` Глобальные конфигурации
- `User.cs` Данные пользователя (уровень, валюта, прогресс)
- `LevelData.cs` Реактивные переменные состояния

#### 🏗️ **Factories** — Фабрики
Создание игровых объектов и сущностей:
- Фабрика врагов
- Фабрика строений
- Фабрика проектилей

#### 📦 **Pools** — Пулы объектов
Управление пулами (ObjectPool) для оптимизации производительности:
- Пул врагов
- Пул проектилей
- Пул эффектов

## 🛠️ Технический стек

- **Unity Engine** — основной фреймворк
- **C#** — основной язык программирования
- **VContainer** — dependency injection контейнер
- **UniRx** — реактивное программирование, reactive extensions for Unity
- **UniTask** — асинхронное программирование
- **MemoryPack** — сериализация данных

## 🏗️ Архитектурные паттерны

### MVC Pattern
- **Model** — данные и состояние (`*Model.cs`)
- **View** — визуальное представление (`*View.cs`)
- **Controller** — логика управления (`*Controller.cs`)

### Dependency Injection (VContainer)
Все зависимости регистрируются и внедряются через контейнер, что обеспечивает слабую связанность кода.
